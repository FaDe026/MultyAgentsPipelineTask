# Career Market Analysis System

Система анализа рынка труда, состоящая из 4 агентов, вызываемых строго последовательно.

## Команда запуска

Для запуска анализа используйте следующую команду, указав необходимую специальность в параметре --role:
```commandline
python main.py --role "Python Developer"
```

## Описание агентов

Система работает по принципу конвейера (pipeline), где каждый следующий агент получает JSON с результатами предыдущего.

### Агент 1: Аналитик рынка (Market Analyst)

Вход: Название специальности (строка).
Выход: skill_map — словарь категорий навыков.
Задачи:
Выделяет hard skills (языки, фреймворки, инструменты) и soft skills.
Для каждого навыка определяет востребованность (critical / important / nice-to-have) и тренд (growing / stable / declining).
Структурирует данные в JSON: {"languages": [...], "frameworks": [...], "infrastructure": [...]}.

### Агент 2: Оценщик зарплат (Salary Estimator)

Вход: skill_map от Агента 1.
Выход: salary_table — вилки зарплат по грейдам и регионам.
Задачи:
Формирует таблицу зарплат для грейдов: Junior / Middle / Senior / Lead.
Разбивает по регионам: Москва / Регионы РФ / Remote USD.
Для каждой ячейки указывает три числа: min, median, max.
Добавляет поле market_trend с обоснованием и список top_employers (3-5 реальных компаний).

### Агент 3: Карьерный советник (Career Advisor)

Вход: skill_map + salary_table от Агентов 1 и 2.
Выход: learning_path, gap_analysis, portfolio_project.
Задачи:
Создает learning_path: 3 фазы по 30 дней (Foundation → Practice → Portfolio). Каждая фаза содержит темы, ≥ 2 ресурса и milestone.
Формирует gap_analysis:
quick_wins: навыки, которые можно освоить за 2-4 недели.
long_term: навыки на 3+ месяца.
Предлагает portfolio_project: конкретную идею проекта с описанием и списком технологий.

### Агент 4: Критик и верификатор (Critic)

Вход: Полный JSON от Агентов 1–3.
Выход: quality_score, warnings, is_consistent.
Задачи:
Проверяет согласованность: соответствуют ли зарплаты уровню навыков.
Ищет противоречия: например, если навык имеет тренд declining, но приоритетен в learning_path — фиксирует предупреждение.
Выставляет quality_score (0-100) с обоснованием.
Возвращает итоговый вердикт is_consistent (true/false).
